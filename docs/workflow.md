# MotionMagic 개발 워크플로우

## 개발 방식

**하이브리드: 로직 우선 + 최소 시각 스캐폴드**

- 핵심 로직을 먼저 구현하되, 단색 사각형 + 텍스트로 최소한의 시각 피드백 제공
- 좌표계는 처음부터 1920×1080 기준으로 확정
- 스프라이트 에셋은 Phase 4에서 교체

---

## Phase 0 — AI 파이프라인 (Phase 1과 병렬 진행)

| 순서 | 작업 | 모듈 |
|------|------|------|
| 0-1 | 웹캠 데이터 수집 (4개 제스처 × 500샘플) | `collector.py` |
| 0-2 | 전처리 (정규화, 증강, 68차원 벡터) | `preprocessor.py` |
| 0-3 | PyTorch Dataset 구현 | `dataset.py` |
| 0-4 | 하이브리드 CNN 학습 | `model.py`, `trainer.py` |
| 0-5 | 규칙 기반 검증 레이어 | `rule_validator.py` |
| 0-6 | 에임 좌표 매핑 & EMA 스무딩 | `aim_tracker.py` |
| 0-7 | 실시간 인식 엔진 통합 | `recognizer.py` |

**검증**: `pytest` + `scripts/evaluate_model.py` (혼동 행렬, 학습 곡선)

## Phase 1 — 게임 코어 (Phase 0과 병렬 진행)

| 순서 | 작업 | 모듈 |
|------|------|------|
| 1-1 | 플레이어 엔티티 (파란 사각형, 화면 내 이동) | `player.py` |
| 1-2 | 적 엔티티 (빨간 사각형, 좌측으로 접근) | `enemy.py` |
| 1-3 | 자동 공격 시스템 (가장 가까운 적에게 투사체) | `combat.py`, `projectile.py` |
| 1-4 | 웨이브 기반 적 스폰 | `spawner.py` |
| 1-5 | 전장 2개 전환 (키보드 Tab) | `battle.py` |
| 1-6 | HUD (텍스트: HP, 웨이브 번호) | `hud.py` |

- 제스처 입력은 **키보드로 대체** (숫자 키 1~4 → fist/palm/point/scissors)
- **검증**: 게임 실행 + `pytest` (데미지 계산, 스폰 로직)

## Phase 2 — 마법 시스템

| 순서 | 작업 | 모듈 |
|------|------|------|
| 2-1 | 제스처 2개 조합 → 마법 매핑 | `magic.py` |
| 2-2 | 마법별 투사체/광역기 구현 | `projectile.py` |
| 2-3 | 적 기믹 구현 (서브클래스) | `enemy.py` |
| 2-4 | 전투 시스템 통합 (자동공격 + 마법) | `combat.py` |

- 여전히 키보드 입력 + 단색 사각형
- **검증**: `pytest` (fist+palm → 파이어볼 등 조합 검증)

## Phase 3 — AI ↔ 게임 통합

| 순서 | 작업 | 모듈 |
|------|------|------|
| 3-1 | 웹캠 캡처 + 추론 백그라운드 스레드 | `camera_thread.py` |
| 3-2 | GestureEvent → Pygame 이벤트 큐 연결 | `gesture_event.py` |
| 3-3 | 에임 크로스헤어 렌더링 | `crosshair.py` |
| 3-4 | 키보드 대체 입력 제거 (제스처 전용 모드) | `battle.py` |

- **검증**: 실제 웹캠으로 플레이테스트 (조명, 거리 등 다양한 환경)
- ⚠️ 리스크 최대 — 스레딩 + 이벤트 큐 통합은 사람의 검증 필수

## Phase 4 — 그래픽 & 폴리시

| 순서 | 작업 | 모듈 |
|------|------|------|
| 4-1 | 사각형 → 스프라이트 에셋 교체 | `entities/` |
| 4-2 | 파티클 이펙트 시스템 | `particles.py` |
| 4-3 | 타이틀 / 결과 화면 | `title.py`, `result.py` |
| 4-4 | 사운드 효과 | `assets/sounds/` |
| 4-5 | 게임 밸런싱 | `settings.py` |

- 사람의 미적 판단이 핵심 — AI 에이전트에게 구체적 시각 명세 필요

---

## Phase 흐름

```
Phase 0 (AI 파이프라인) ──────────────────┐
                                          ├──→ Phase 3 (통합) ──→ Phase 4 (폴리시)
Phase 1 (게임 코어) ──→ Phase 2 (마법) ───┘
```

- Phase 0 + Phase 1은 **병렬 진행** 가능 (독립적)
- Phase 3 진입 전에 Phase 0, 2 모두 완료 필요
